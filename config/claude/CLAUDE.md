# Global Instructions

## Git Commits

Always use conventional commit syntax for commit messages. Format:

```
<type>(<optional scope>): <description>

<optional body>
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`, `revert`

## Pull Requests

When opening a PR, always check for and use the repo's PR template if one is present (e.g., `.github/pull_request_template.md`).
